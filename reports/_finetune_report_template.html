Executive Summary

Three image classifiers — ResNet50, ViT-Base, and DiT (Document Image Transformer) — were re-fine-tuned on a corpus that nearly doubled in size (1,014 → 1,873 pages) and was rebalanced so that handwritten and questionnaire-style pages (the failure mode we care about) are well represented. Splits were regrouped by source_doc_stem to eliminate page-level leakage, BCE loss is now weighted to offset the reversed class imbalance, and the final operating thresholds were retuned with the 10·FN + FP cost model described above.

On the internal held-out test set all three models reach F1 > 0.95 and ROC-AUC > 0.99. At the new cost-weighted threshold τ* all three hit 0 FN on the test split. Key Methodology Changes vs Phase 5

  • Split key: template_family → source_doc_stem (54 unique source docs) to prevent page-level leakage for multi-page source PDFs.
  • Loss weighting: BCEWithLogitsLoss(pos_weight = n_neg / n_pos) so the reversed imbalance (now ~1:1.95) doesn't overpower the safe class.
  • Sampler: optional WeightedRandomSampler upsamples the minority class every epoch (enabled for all three runs).
  • Rubric scoring: incremental Claude vision annotation of the 925 new pages; 1,873 total pages now carry D/H/S/L rubric scores.
  • Cost-weighted threshold: per-model τ* from TemperatureCalibrator.get_cost_weighted_thresholds(fn_cost=10, fp_cost=1), replacing the old "FSR ≤ 5% target" threshold rule.
  • No architecture changes — only data, loss, sampler, and threshold policy.

Headline Findings

Cost-weighted τ* reduces FSR by 9–30 pp on the external set across all three models. DiT goes from 31.9% FSR to 2.1%; ResNet50 from 11.7% to 3.2%; ViT-Base from 26.6% to 9.6%. The review queue grows in exchange (accuracy at τ* falls into the 50–70% range) but that is the correct operating point given the FN = 10× FP cost assumption.
The template shortcut is broken. Adding 6 new source folders (handwritten edge cases, full questionnaires, the special-font corpus) forced the models to learn content-level cues instead of institution-level cues. On the source-doc-grouped test split all three models reach F1 > 0.95 and ROC-AUC > 0.99.
Expected accuracy drop at τ* is correct behaviour. The cost model explicitly trades FP for FN. With τ* pulled well below 0.5, a large fraction of borderline safe documents now route to human review. For a safety-critical classifier whose downstream consumer auto-extracts text, this is the right trade.

Recommendation

Promote DiT to production at the new cost-weighted threshold. DiT has the lowest external FSR (2.1%) of the three models, the highest external accuracy at τ* (66–68%), and its stored T_low / T_high already reflect the new policy.

  • p < T_low = τ* → safe — auto-extract
  • T_low ≤ p < T_high → review queue
  • p ≥ T_high → high risk — reject or escalate

Keep ResNet50 as a secondary / ensemble candidate; it has the highest test-set ROC-AUC (0.999) and is much cheaper to run, so pairing it with DiT as a two-stage filter is worth exploring. ViT-Base trails both on external-set FSR and accuracy and has no clear production niche.

Next Steps

  1. Ensemble (ResNet50 + DiT) with the cost-weighted τ* applied to the averaged probability — expect FSR < 2% with a smaller review queue than DiT alone.
  2. Expand the external validation set beyond 400 documents to reduce variance on FSR estimates (94 risky docs give ~1.4 pp binomial std at the current 2% FSR level).
  3. Add per-source-folder FSR tracking to the production monitoring dashboard — this is the metric that told us the shortcut was there.
  4. Revisit the FN:FP cost ratio with the downstream team. A 10:1 ratio floods reviewers; 5:1 or 3:1 may be defensible if downstream hallucination checks exist.
